Use CocoaPods Dependencies in RubyMotion Apps
=============================================
Laurent Sansonetti <lrz@hipbyte.com>

이 글은 RubyMotion프로젝트에서 의존성 관리 도구인 http://cocoapods.org[CocoaPods] 를 어떻게 사용할지를 다룹니다.

Synopsis
--------

CocoaPods는 Objective-C의 의존성 관리 도구입니다. Objective-C 의존성을 위한 RubyGems, Bundler라고 보셔도 무방합니다.

CocoaPods안에서 의존성은 Pods라고 불립니다. GitHub 저장소 http://github.com/CocoaPods/Specs[CocoaPods/Specs] 에서 사용가능한 Pods의 목록을 보실 수 있습니다.

CocoaPods는 원래 Objective-C Xcode를 위해 설계되었습니다만, 멋진 CocoaPods 개발자와 함께 CocoaPods를 RubyMotion프로젝트에서 사용할 수 있게 만들었습니다.

Installation
------------

먼저 CocoaPods를 설치하고 설정해야 합니다. http://guides.cocoapods.org/using/getting-started.html#installation[installation guide] 의 CocoaPods의 설치법을 참고해 주십시오. 설치하셨다면 다음을 실행합니다.

----
$ pod setup
----

http://github.com/HipByte/motion-cocoapods[RubyMotion CocoaPods integration] 을 사용해 gem으로서 설치하는 것도 가능합니다.

----
$ sudo gem install motion-cocoapods
----

Usage
-----

프로젝트의 'Rakefile' 안에 gem을 require해야 합니다.

----
require 'rubygems'
require 'motion-cocoapods'
----

이제 애플리케이션 설정에서 Podfile을 사용 하던 것 처럼 간단히 +pods+ 메소드를 사용해 require할 Pods를 적을 수 있습니다.

예를들어 +AFNetworking+ Pod를 require하고 싶으시다면,

----
Motion::Project::App.setup do |app|
  # ...
  app.pods do
    pod 'AFNetworking'
  end
end
----

이제, 소스를 받고 다음 빌드부터 프로젝트 설정에 +AFNetworking+ 을 추가하는 CocoaPods integration을 실행하면 됩니다.

----
$ rake pod:install
----

이제 프로젝트를 빌드할 때, +AFNetworking+ 소스도 어플리케이션 실행파일과 함께 빌드, 링크 될 것입니다.
